This method copies data using multiple requests so large objects can be copied with a normal length timeout per request rather than one very long timeout for a single request.

    'encryptionKey' => $key,

signedUploadUrl()

StorageObject.php : 940

Create a Signed Upload URL for this object.

public signedUploadUrl(Timestamp|DateTimeInterface|int $expires[, array<string|int, mixed> $options = [] ]) : string

This method differs from StorageObject::signedUrl() in that it allows you to initiate a new resumable upload session. This can be used to allow non-authenticated users to insert an object into a bucket.

In order to upload data, a session URI must be obtained by sending an HTTP POST request to the URL returned from this method. See the Cloud Storage Documentation for more information.

If you prefer to skip this initial step, you may find StorageObject::beginSignedUploadSession() to fit your needs. Note that beginSignedUploadSession() cannot be used with Google Cloud PHP's Signed URL Uploader, and does not support a configurable expiration date.

Example:

$timestamp = new Timestamp(new \DateTime('tomorrow'));
$url = $object->signedUploadUrl($timestamp);
