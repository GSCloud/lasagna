image()

VisionClient.php : 148

Create an instance of {@see Google\Cloud\Vision\Image} with required features and options.

public image(resource|string|StorageObject $image, array<string|int, mixed> $features[, array<string|int, mixed> $options = [] ]) : Image

This method should be used to configure a single image, or when a set of images requires different settings for each member of the set. If you have a set of images which all will use the same settings, VisionClient::images() may be quicker and simpler to use.

This method will not perform any service requests, and is meant to be used to configure a request prior to calling VisionClient::annotate().

For more information, including best practices and examples detailing other usage such as $imageContext, see Image::__construct().

Example:

$imageResource = fopen(__DIR__ . '/assets/family-photo.jpg', 'r');

$image = $vision->image($imageResource, [
    'FACE_DETECTION'
]);

// Setting maxResults for a feature

$imageResource = fopen(__DIR__ . '/assets/family-photo.jpg', 'r');

$image = $vision->image($imageResource, [
    'FACE_DETECTION'
], [
    'maxResults' => [
        'FACE_DETECTION' => 1
    ]
]);
Parameters

An image to configure with the given settings. This parameter will accept a resource, a string of bytes, the URI of an image in a publicly-accessible web location, or an instance of StorageObject.

$features : array<string|int, mixed>

images()

VisionClient.php : 197

Create an array of type {@see Google\Cloud\Vision\Image} with required features and options set for each member of the set.

public images(array<string|int, resource>|array<string|int, string>|array<string|int, StorageObject> $images, array<string|int, mixed> $features[, array<string|int, mixed> $options = [] ]) : array<string|int, Image>

This method is useful for quickly configuring every member of a set of images with the same features and options. Should you need to provide different features or options for one or more members of the set, VisionClient::image() is a better choice.

This method will not perform any service requests, and is meant to be used to configure a request prior to calling VisionClient::annotateBatch().

For more information, including best practices and examples detailing other usage such as $imageContext, see Image::__construct().

Example:

// In the example below, both images will have the same settings applied.
// They will both run face detection and return up to 10 results.

$familyPhotoResource = fopen(__DIR__ . '/assets/family-photo.jpg', 'r');
$weddingPhotoResource = fopen(__DIR__ . '/assets/wedding-photo.jpg', 'r');

$images = $vision->images([$familyPhotoResource, $weddingPhotoResource], [
    'FACE_DETECTION'
], [
    'maxResults' => [
        'FACE_DETECTION' => 10
    ]
]);
Parameters

$features : array<string|int, mixed>
